# Task RFP-M17: Single canonical changelog (consolidation)

**Missing idea:** M17  
**RFP deliverable:** Desired outcome + (i) Present New Documentation Strategy

---

## Objective

Consolidate **multiple** changelogs into **a single canonical feed** and document owners, so the RFP requirement is met (not only a placeholder changelog).

## Scope

- Desired outcome + (i): “Consolidation of multiple changelogs into **a single canonical feed**”
- Report: changelog is placeholder; “consolidation” of multiple changelogs into one is not verified
- Deliverable: identify existing changelogs, choose or create the canonical feed, migrate or link, and assign owners

## Deliverables

- Inventory: list of current changelog sources (e.g. GitHub releases, Notion, Studio, product-specific)
- Canonical changelog: one place (e.g. docs changelog page, Mintlify changelog, or GitHub Releases) designated as “the” docs/product changelog with a clear URL
- Plan or execution: how other changelogs feed into it (manual copy, automation, or “deprecated in favor of X”)
- Owners: who is responsible for updating the canonical changelog
- Update gaps report so M17 is marked complete when the single feed exists and is documented

## Acceptance criteria

- [ ] One canonical changelog feed is designated and linked from the docs
- [ ] Multiple previous sources are either consolidated into it or explicitly deprecated in its favor
- [ ] Ownership is assigned and documented
- [ ] RFP gap M17 is marked complete in the gaps report
