# Task RFP-M15: Ticketing system (clear, for reporting and patching)

**Missing idea:** M15  
**RFP deliverable:** (iv) Public Workflow For Maintenance & Community Contributions

---

## Objective

Provide a **clear ticketing system** for reporting problems and patching fixes: documented process + visibility, not only “GitHub issues exist.”

## Scope

- RFP (iv): “Provide a **clear ticketing system** for reporting problems and patching fixes”
- Report: “GitHub issues exist, but no documented ticketing/triage process.” The RFP asks for a *system* (process + visibility)
- Deliverable: document how to report docs problems, how they are triaged, and how patches are submitted (e.g. issue templates, labels, SLAs, link from docs)

## Deliverables

- Ticketing system doc: where to report (e.g. GitHub repo + label, or Discord channel), how to triage (labels, assignee), and how to patch (PR workflow, link to contribution guide)
- Optional: GitHub issue template(s) for “docs bug” and “docs fix”; labels (e.g. `docs`, `area: gateways`)
- Link from docs (e.g. contribute page, footer) to “Report a docs problem” and “Propose a fix”

## Acceptance criteria

- [ ] Process is written down: report → triage → patch
- [ ] Users can find “where to report” and “how to propose a fix” from the docs or repo
- [ ] RFP gap M15 is marked complete in the gaps report
