## Objective

Treat “incorporate starter repos” as a distinct RFP deliverable: identify official or recommended starter repos, add them to the docs in a findable way, and track completion.

## Scope

- RFP (ii): “Work with existing groups to incorporate **starter repos**, examples, copy-paste snippets…”
- Task list has “incorporate starter repos”; report says “not systematically organized”
- Deliverable: a curated set of starter repos (per persona or use case) with clear doc integration

## Deliverables

- List of starter repos to incorporate (with owner, repo URL, and target persona or use case)
- Docs integration: dedicated “Starter repos” or “Examples” hub/section, or consistent placement in quickstarts/developer guides with links
- Update task list / report so “starter repos” is a checked deliverable

## Acceptance criteria

- [ ] At least one starter repo per major developer path (e.g. AI Jobs, Transcoding/streaming) is linked and described in the docs
- [ ] Links are in a consistent place (e.g. developer hub, quickstart, or examples page)
- [ ] RFP gap M7 is marked complete in the gaps report when done
