# Task RFP-M1: Identify core stakeholder groups by name

**Missing idea:** M1  
**RFP deliverable:** (i) Present New Documentation Strategy

---

## Objective

Produce a single, explicit list of core stakeholder groups for the documentation restructure (Foundation, Inc, AI SPE, Cloud SPE, Streamplace, Frameworks, Daydream, etc.) and map it into a deliverable artifact so it is tracked and reviewable.

## Scope

- RFP requirement: “Identify core stakeholder groups (Livepeer Foundation, Livepeer Inc, AI SPE, Cloud SPE, Streamplace, Frameworks and more)”
- Notion has “Stakeholder Categories” and “Key Stakeholders” DBs but no single RFP deliverable that names these groups
- Task list has “Identify stakeholders for content requirements” without the explicit list

## Deliverables

- One artifact (doc or Notion page) that lists core stakeholder groups by name with a short role/scope note per group
- Optional: mapping from this list to existing Notion stakeholder DBs or to doc sections/personas
- Update task list / Planning Overview to reference this artifact

## Acceptance criteria

- [ ] All of: Livepeer Foundation, Livepeer Inc, AI SPE, Cloud SPE, Streamplace, Frameworks (and any other named in RFP) are explicitly listed
- [ ] Daydream and other “and more” groups are either listed or explicitly scoped as out-of-scope
- [ ] Artifact is linked from `docs-v2-rfp-task-list-and-plan.md` or Notion Planning Overview
