# Task RFP-M8: Realtime API coverage (explicit)

**Missing idea:** M8  
**RFP deliverable:** (ii) Re-Write Documentation

---

## Objective

Explicitly verify and document **realtime** API coverage in the docs so that “full API/SDK/CLI references with updated coverage (including **realtime** + BYOC APIs)” is complete for realtime.

## Scope

- RFP (ii): “full API/SDK/CLI references with updated coverage (including **realtime** + BYOC APIs)”
- BYOC is covered; realtime API coverage is not explicitly verified or listed as done/not done
- Deliverable: audit realtime APIs, add or fix references, and record status

## Deliverables

- Audit: list of realtime APIs/SDK surfaces that should be documented (from product/specs)
- Docs: ensure realtime APIs have reference pages or sections (or a clear “realtime API” index)
- One-line status in gaps report or task list: “Realtime API coverage: [done/partial with link]”

## Acceptance criteria

- [ ] Realtime API surface is identified (from gateway/product docs or eng)
- [ ] Documentation includes realtime API references or a clear pointer to where they live
- [ ] RFP requirement “realtime + BYOC” is explicitly marked complete for realtime (or scoped out with reason)
