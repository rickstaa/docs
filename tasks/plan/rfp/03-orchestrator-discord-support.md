**Missing idea:** M3  
**RFP deliverable:** (i) Present New Documentation Strategy

---

## Objective

Create a documented **plan** for ongoing Discord support for orchestrators (not only “Discord escalation” in setup), so that the RFP requirement “plan for support in Discord” is satisfied.

## Scope

- RFP (i): “Orchestrators: simplify documentation to easy onboarding with **plan for support in Discord**”
- Phase 3 mentions “Discord escalation” in orchestrator setup but not a **plan** for ongoing support
- Deliverable: a short, written plan (where to ask, who responds, SLAs or expectations, link from orchestrator docs)

## Deliverables

- A “Discord support plan” section or doc: how orchestrators get help in Discord (channel(s), roles, escalation, any SLAs)
- Link from orchestrator docs (e.g. `v2/pages/05_orchestrators/`) to this plan
- Optional: Notion/Forum update so this is tracked as an RFP deliverable

## Acceptance criteria

- [ ] Written plan exists and is linked from orchestrator documentation
- [ ] Plan covers where to ask, who responds (or how to find help), and any escalation path
- [ ] RFP gap M3 can be marked complete in the gaps report when this is done
